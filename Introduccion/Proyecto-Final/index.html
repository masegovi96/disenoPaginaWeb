Proyecto Final

Portafolio Web Personal

Demuestra tus habilidades en diseño web, maquetación y desarrollo frontend creando un producto profesional que puedas utilizar en tu vida laboral.

Objetivo

Desarrollar un sitio web personal tipo portafolio donde el estudiante demuestre sus habilidades en diseño web, maquetación y desarrollo frontend, creando un producto profesional que pueda utilizar en su vida laboral.

Descripción General

Cada alumno desarrollará de forma individual un sitio web que funcione como su portafolio personal. El proyecto deberá cumplir con lo siguiente:

Diseñado previamente

Presenta un wireframe o mockup antes de comenzar a codificar.

Tecnologías de clase

Desarrollado con las tecnologías y herramientas vistas durante el cuatrimestre.

Responsive

El sitio debe adaptarse correctamente a dispositivos móviles.

Publicado en internet

El proyecto debe estar accesible públicamente en línea.

Respaldado en GitHub

Debe subirse y mantenerse en tu repositorio personal con múltiples commits.

Contenido Mínimo del Sitio

Inicio

  • Nombre completo
  • Rol profesional
  • Imagen o avatar
  • Mensaje de presentación

Sobre mí

  • Descripción personal / profesional
  • Habilidades
  • Tecnologías que maneja

Proyectos

Mínimo 2–3 proyectos, cada uno con:

  • Imagen representativa
  • Descripción
  • Tecnologías utilizadas

Contacto

  • Formulario con validación en JavaScript
  • Redes sociales

Requisitos Técnicos

HTML5 semántico
CSS3 — Flexbox y/o Grid
Uso de Bootstrap
JavaScript básico — interactividad
Diseño responsive — mobile-first
Código ordenado y bien estructurado

Publicación (Obligatorio)

El sitio deberá estar publicado en internet mediante alguna de estas opciones:

GitHub Pages
Netlify
Vercel

No se aceptan proyectos solo en local

Repositorio (Requisito clave)

Cada alumno deberá utilizar su cuenta personal de GitHub y cumplir con:

  • Crear un repositorio personal
  • Subir el proyecto en una carpeta independiente del resto de trabajos
  • Mantener orden en archivos y carpetas
  • Realizar múltiples commits (no subir todo en un solo intento)

Presentaciones del Proyecto

Presentación Final Obligatoria

Se realiza el día del examen final. El alumno deberá:

  • Mostrar el sitio en funcionamiento
  • Explicar su estructura
  • Justificar su diseño
  • Demostrar funcionalidades

Presentación Tipo Evento Vespertino

No es obligatoria, pero es altamente recomendable. Sus objetivos:

  • Simular un entorno profesional real
  • Practicar exposición ante público real
  • Recibir retroalimentación externa
  • Asistencia de personas externas

Condiciones Importantes

No se aceptan proyectos con alguna de las siguientes condiciones:

Proyectos incompletos
Sitios no funcionales
Código desordenado
Páginas no responsive
